# 首页 UI 规格

## 页面任务

首页负责完成三件事：

- 让访客快速判断这件事和自己有没有关系
- 让访客理解“扫码增长”不是活动页，而是长期经营入口
- 把访客导向解决方案、案例或商务沟通

## 视觉关键词

- 强销售感
- 明亮、清晰、可信
- 包装扫码与增长结果并置
- 紫蓝渐变强调，不要过度科技感

## 页面结构

1. Hero  
左文右图。左侧大标题、副标题、双 CTA；右侧放“包装 + 手机会员界面 + 数据路径”主视觉。

2. 数据证明横带  
放 3 到 4 组关键数字或品牌结果，占满整行，背景可用浅紫渐变。

3. 问题段  
讲“流量不一定最贵，留不下来才更贵”，图文交替。

4. 方案段  
讲“扫码是关系入口，不只是领取动作”，图文交替。

5. 结果段  
讲“活动之后的留存、活跃和复购”，图文交替。

6. 适配品牌段  
讲“哪些品牌更值得先做”，可用图文 + 行业标签混排。

7. 轻案例入口  
用 2 到 3 张案例卡做快速跳转。

8. CTA 收口  
强 CTA 区，强调“先判断是否适合，再决定从哪里开始”。

## 版式重点

- Hero 高度建议 `min-height: 78vh`
- Hero 左右比例建议 `1fr / 0.95fr`
- 数据横带独立出一屏半的节奏感，不要缩成普通卡片
- 内容段采用左右交替图文，避免首页出现三等分卡片墙
- 轻案例入口做非对称 2+1 布局，不做四宫格

## 交互与动效

- 顶部固定 progress bar
- Hero 按钮 hover 上浮
- 内容段滚动 reveal
- 数据数字滚动计数
- 行业标签 hover 轻高亮

## 占位图清单

1. Hero 主视觉  
用途：首屏右侧核心图  
比例：`16:9`
- 推荐尺寸：`1920 × 1080px`
- 建议文件名：`home-hero-main.jpg`

2. 问题段配图  
用途：表达“流量分散、活动之后没有后续”  
比例：`4:3`
- 推荐尺寸：`1440 × 1080px`
- 建议文件名：`home-problem-scene.jpg`

3. 方案段配图  
用途：表达“包装扫码进入会员承接”  
比例：`4:3`
- 推荐尺寸：`1440 × 1080px`

4. 结果段配图  
用途：表达“会员活跃、内容触达、复购提醒”

- 推荐尺寸：`1440 × 1080px`
