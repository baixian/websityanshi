# 解决方案页 UI 规格

## 页面任务

解决方案页负责讲清楚业务逻辑：品牌如何从一次触达走到留存、活跃和复购。

## 视觉关键词

- 路径感
- 阶段感
- 结构清晰
- 比首页更理性、更解释性

## 页面结构

1. Hero  
左文右图。右侧不是产品界面，而是“增长路径总览图”。

2. 触达承接段  
说明不同入口如何进入统一承接路径。

3. 关系沉淀段  
说明如何从扫码用户变成会员关系。

4. 后续运营段  
说明内容、权益和节点活动如何继续配合。

5. 试点放大段  
说明如何从重点产品线、重点渠道开始，再复制放大。

6. 方案证明区  
放路径图、过程型指标或方法说明。

7. CTA 区  
引导用户进入更具体的方案沟通。

## 版式重点

- 整页更强调纵向叙事，不强调大卡片装饰
- 每段左侧文案、右侧示意图，下一段反过来
- 过程图尽量用线性路径、节点、箭头和阶段编号
- 证明区建议做成横向时间轴或路径闭环图

## 交互与动效

- 顶部 progress bar
- 每段标题和图同步 reveal
- 路径图节点进入视口时逐一显现
- CTA 区按钮 hover 上浮
