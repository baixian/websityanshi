# 客户案例页 UI 规格

## 页面任务

客户案例页负责建立结果证明，让访客看到“原来卡在哪里、后来怎么改善”。

## 视觉关键词

- 证明感
- 结果感
- 项目感
- 比其他页更强调前后变化和指标

## 页面结构

1. Hero  
讲“案例真正有价值的是改变了什么”。

2. 重点案例首屏卡  
展示 1 个代表案例，带项目标签和核心结果。

3. 新品推广案例段  
强调首波流量后续沉淀。

4. 门店动销案例段  
强调线下触点后的再沟通与再购买。

5. 渠道活动案例段  
强调多区域、多终端的复制方法。

6. 会员运营案例段  
强调长期陪伴、活跃和复购。

7. 证明区  
用结果句、指标块或品牌 Logo 墙收口。

8. CTA 区  
引导预约案例沟通。

## 版式重点

- 可做 `1 个重点案例大卡 + 3 个分段案例`
- 每个案例都建议有：问题、动作、变化、结果标签
- 指标卡和 Logo 可以穿插，但不要变成表格式报告页
- 卡片比产品页更厚重一点，增强“项目证明”气质

## 交互与动效

- 顶部 progress bar
- 案例卡片 reveal
- 指标卡 hover 轻抬
- 重点案例卡做更明显的阴影和渐变边缘
