# 安全保障页 UI 规格

## 页面任务

安全保障页负责建立信任感，说明为什么这件事可以放心推进。

## 视觉关键词

- 稳重
- 清晰
- 有秩序
- 较少营销感，更多边界感和规范感

## 页面结构

1. Hero  
直接讲“做增长前，先把边界和保障看清楚”。

2. 数据处理段  
强调清晰边界和规范处理。

3. 权限分工段  
强调谁来配置、谁来查看、谁来执行。

4. 活动风控段  
强调奖励发放、核销和异常处理。

5. 服务支持段  
强调上线支持、问题响应和阶段复盘。

6. 证明区  
用保障清单、流程图或服务响应结构收口。

7. CTA 区  
引导用户先了解保障点再进入合作沟通。

## 版式重点

- 不做夸张渐变和大面积悬浮卡
- 多用说明块、规则卡、清单结构
- 保持浅底，但可加入更理性的蓝灰色块
- 图形更偏权限矩阵、流程关系图、规范示意

## 交互与动效

- 顶部 progress bar
- 清单项 reveal
- 卡片 hover 轻边框亮起，不做明显弹跳
- 动效整体减弱，比首页和产品页更克制
