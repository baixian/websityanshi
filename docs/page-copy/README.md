# 官网文案方向

本目录用于沉淀各页面的正式文案稿，供页面改版、设计联调和后续替换真实案例数据时直接使用。

## 这轮改稿目标

- 去掉“像 AI 在解释方案”的语气，改成更像官网对外表达的语气。
- 减少抽象大词和模板句，多用客户能直接听懂的业务表达。
- 让每个页面各讲一件事，避免 8 个页面反复说同一套话。
- 让标题更短、更稳、更像官网，而不是一整句说明文。

## 统一写法

- 主标题负责立住价值判断，尽量短，尽量有记忆点。
- 副标题负责补充业务场景、适用对象和能带来的变化。
- 每一段只讲一个价值点，不在同一段里同时解释太多概念。
- 优先写客户会关心的结果，如留资、活跃、复购、协同、推进效率。
- 正文先说业务问题，再说页面价值，最后再落到行动判断。

## 交付口径

- 主标题尽量控制在 8 到 16 个字之间。
- 副标题尽量控制在 1 到 2 句之间，不写成长说明文。
- 收益点优先写结果本身，少连续使用 `让 / 提升 / 更容易` 这类相同句式。
- CTA 尽量写成具体动作，如 `获取建议 / 查看案例 / 预约演示`，不只写泛泛的“了解更多”。

## 去 AI 味规则

- 少用高频模板句：`围绕……帮助……让……`、`不是……而是……`、`重点讲清……`、`先……再……` 连续重复出现。
- 少用抽象结论词：`赋能`、`闭环`、`全链路`、`数字化引擎`、`能力矩阵`。
- 少用解释腔：不要总在文案里解释“这个页面想说明什么”。
- 少用方法论腔：不要把官网写成培训材料或咨询报告。
- 多用场景词和结果词，让访客一眼能代入自己的业务。

## 页面分工

- 首页：先让访客判断这件事和自己有没有关系。
- 解决方案页：讲清楚这条路径怎么跑通。
- 产品支持页：讲清楚系统怎么把动作接起来。
- 行业场景页：帮助不同行业快速对号入座。
- 客户案例页：用结果变化建立信任。
- 合作方式页：帮助判断现在适合怎么合作。
- 安全保障页：解决“能不能放心推进”的顾虑。
- 商务咨询页：把沟通门槛降下来，把下一步讲明白。

## 禁用倾向

- 过度技术化：如“接口”“数据回流”“自动化链路”“SCRM 沉淀”。
- 过度口语化：如“扫完就走”“先试试看”“后面没接住”。
- 过度空泛化：如“赋能增长”“全域运营”“长期价值引擎”。
- 直接冒用他站真实客户、真实数据、真实服务承诺作为本站事实。

## 可借鉴但不可直接冒用

- 可借鉴：案例组织方式、结果表达方式、服务介绍结构、CTA 节奏。
- 不可直接冒用：他站客户名称、项目结果、公司地址、联系电话、邮箱、服务承诺。
- 正确做法：先按成熟官网结构写出本站终稿，再把本站真实客户、真实项目、真实数据替换进去。

## 内部占位使用规则

- 如需暂时借用他站客户名、数据和服务信息，只能在内部文案稿中以 `参考占位` 形式出现。
- 所有占位内容都必须带上 `上线前替换` 标记。
- 这类内容只能用于内部排版、设计和演示，不应直接作为对外官网事实发布。
