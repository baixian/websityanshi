# 客户案例页文案稿

## 页面定位

通过案例说明不同场景下“原来卡在哪里、后来怎么改善”，重点呈现前后变化和业务结果，而不是堆项目名称。

## 建议案例成稿格式

每个案例最终建议统一为：

1. 品牌 / 项目名称
2. 场景说明
3. 用户参与动作
4. 品牌提供的权益或互动
5. 带来的业务变化

## 第一段：新品推广案例

- 标题：新品上市不只要有声量，还要留得住第一批人
- 副标题：很多新品推广在首波传播后就迅速降温，案例里更值得看的，是品牌如何把尝鲜用户接进后续触达，让热度不只停留在首发阶段。
- 收益点：
  - 提升新品活动后的留资和留存空间
  - 提升尝鲜用户进入会员连接的机会
  - 提升新品推广投入的长期积累价值

## 第二段：门店动销案例

- 标题：门店动销不该只停留在当次成交
- 副标题：门店活动往往最接近真实消费现场，因此案例里更关键的，不是当场核销了多少，而是用户离店之后品牌还能不能继续触达和带回购。
- 收益点：
  - 放大门店活动后的后续价值
  - 提升线下触点后的再沟通机会
  - 提升到店用户再次购买的可能性

## 第三段：渠道活动案例

- 标题：渠道活动越多，越需要一套能复制的方法
- 副标题：当品牌同时面对不同区域、不同终端和不同合作伙伴时，案例真正能说明问题的，是哪些动作值得复制，哪些结果能够稳定复现。
- 收益点：
  - 帮助渠道活动更容易被统一复盘
  - 帮助终端触点更容易转成用户连接
  - 帮助品牌逐步沉淀更稳定的方法论

## 第四段：会员运营案例

- 标题：复购提升，往往来自长期陪伴而不是单次刺激
- 副标题：案例里最能说明问题的，不是某一次优惠带来多少成交，而是品牌如何通过会员连接、内容触达和节日互动，让用户持续回来。
- 收益点：
  - 让会员关系更有活跃度和粘性
  - 让后续触达更贴近用户购买节奏
  - 让复购提升建立在连续运营之上

## 证明区

- 标题：案例的意义，在于帮品牌更快判断什么对自己有用
- 副标题：看清原来卡在哪里、后来怎么改善，比单纯罗列项目数量更能帮助品牌找到适合自己的做法。

## 建议证据素材

- 每个案例至少补 1 个“原来问题”与 1 个“后来的变化”
- 每个案例尽量补 1 到 2 个结果指标：如 `留资提升 / 活跃提升 / 复购提升 / GMV 增长`
- 建议补品牌名、行业、项目周期和核心动作，避免案例只剩抽象结论
- 建议每个案例补 1 句“用户参与动作”：如 `扫码、签到、分享、积分、抽奖、兑换、排名、体验官申请`
- 建议每个案例补 1 句“品牌提供的权益”：如 `积分、礼包、体验装、活动资格、会员权益`
