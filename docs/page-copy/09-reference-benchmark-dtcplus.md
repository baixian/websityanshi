# 得效案例页借鉴说明

## 页面来源

- [得效典型案例](http://www.dtcpluscrm.com/classic-case.html)

## 可以直接借鉴的表达结构

### 1. 案例条目结构

它的案例页不是长篇复盘，而是：

- 品牌名
- 项目类型
- 用户参与动作
- 用户能获得什么
- 品牌最终想达成什么

这种结构很适合官网，不会太重，也比单纯写“某品牌案例”更完整。

### 2. 服务信息写法

它的服务信息常见写法是：

- 专属会员中心
- 官方活动小程序
- 日常签到 / 打卡 / 分享 / 扫码 / 排名
- 积分、奖励、体验装、抽奖、兑换
- 用户成为体验官、内容参与者或传播者

这些词说明一个成熟官网在讲案例时，会把“用户做什么”和“品牌得到什么”放在同一句里。

### 3. 结果表达方式

它的结果表达不是后台语言，而是偏对外业务语言，例如：

- 获得积分累计与排名
- 赢得门票与礼包
- 完成拉新
- 浏览产品、活动、购买方式与体验报告

这种写法适合作为我们案例页的“场景结果句”模板。

## 对本站最有用的借鉴点

### 首页

- 可以借它“活动不是终点，用户后续怎么留下来更重要”的表达习惯。
- 可以借它把“用户动作”和“品牌目标”放在同一段里的方式。

### 解决方案页

- 可以借它“从用户参与动作反推经营价值”的讲法。
- 少讲系统，多讲用户参与、积分、互动、内容、留存、复购。

### 产品支持页

- 可以借它按“签到 / 扫码 / 分享 / 排名 / 积分 / 兑换”这种具体运营动作来解释产品能力。
- 让产品介绍更像“运营抓手”，而不是“系统模块”。

### 客户案例页

- 最值得借鉴。
- 建议每个案例都按下面格式写：

1. 品牌 / 项目名称
2. 应用场景
3. 用户参与动作
4. 品牌提供的权益或互动
5. 带来的业务变化

示例模板：

- 标题：某品牌新品推广项目
- 副标题：围绕新品上市活动，用户可通过扫码、签到、互动或分享进入会员体系，完成留资、参与活动并获得相应权益，帮助品牌把首波流量沉淀为后续可持续运营的人群。

### 合作方式页

- 可以借它“会员中心 / 官方活动 / 品牌互动 / 体验官”这些更贴近业务场景的说法。
- 不要只写“试点 / 陪跑 / 协同”，可以补进“适合做什么类型项目”。

## 不建议直接照搬的内容

- 真实客户名称，如 `莎辛那`、`宝洁`、`SeeSaw Coffee`
- 真实系统名，如 `PITAYA WeChat HTML5 Analysis System`
- 真实地址、电话、邮箱
- 没有被本站真实验证的数据和承诺

## 建议替换策略

### 如果暂时没有真实案例名

- 先用行业型标题，不用捏造品牌名。
- 例如：
  - `某快消品牌新品推广项目`
  - `某酒水品牌节庆会员活动`
  - `某连锁零售品牌门店动销项目`

### 如果暂时没有真实结果数据

- 先写“结果方向”，不写虚假数值。
- 例如：
  - `提升首波活动后的留资效率`
  - `增加活动后会员活跃机会`
  - `为后续复购和节日回流打下基础`

### 如果暂时没有完整服务信息

- 先写“品牌能得到什么服务支持”，不要虚构机制。
- 例如：
  - `围绕品牌活动目标提供项目策划、页面承接、用户互动与阶段复盘支持`
  - `结合产品线、门店场景和活动节奏制定更合适的推进方式`

## 结论

这个案例页最值得借的，不是它的客户名，而是它的表达方式：

- 用具体品牌场景讲价值
- 用用户动作讲项目内容
- 用业务结果讲案例意义

这套结构已经适合继续用于本站官网终稿优化。
